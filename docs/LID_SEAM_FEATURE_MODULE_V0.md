# Lid Seam Feature Module v0

Date: 2026-06-30

## Result

Lid Seam v0 adds exactly one visible feature after Box Primitive: a pure-clay
lid seam marker for a preview Lidded Box fixture.

This is not a crate, trim band, panel set, handle, latch, material look,
UV/texturing feature, rig, animation, or Family Studio flow.

## Product Shape

User-facing object:

```text
Lidded Box
```

Copy:

```text
A simple box with a visible lid seam.
```

The visible control is:

```text
Lid Seam
```

The internal control ID remains `lid_height` so the feature module can keep a
stable contract while the user-facing label stays honest.

## Internal Contract

The `lid-seam` feature module declares:

- requires a closed box body
- requires a top/lid candidate zone
- provides a visible `lid_seam` role
- owns the `lid_height` control
- provides lidded-box candidate hooks
- gates seam visibility, non-material appearance, box silhouette preservation,
  and endpoint readability

The module contract lives in `shape-foundry` and rejects nullable-only modules
without behavior hooks.

## Geometry

The v0 seam is export-safe clay geometry:

- the Box Primitive body remains a closed rounded box
- the lidded preview adds one shallow raised clay seam marker
- the seam marker varies by length and weight through the `Lid Seam` control
- the seam is not a texture, material stripe, decal, trim band, latch, handle,
  vent, or panel field

This v0 intentionally does not try to solve a full wraparound lid system. That
belongs in a later gate only if this Lidded Box Make baseline passes.

## Candidate Ideas

These are lidded box ideas only. They do not claim crate/case status.
