# Control Endpoint Visibility Reports

Prompt 2 adds endpoint visibility reporting for the three starter templates.
The report is generated by
`generate_foundry_control_endpoint_visibility_report` in `shape-search`.

## Classification

Endpoint rows use these classes:

- `Strong`
- `Clear`
- `SubtleButExplainable`
- `TooSubtle`
- `Unsupported`

Every visible, non-runtime control receives a row. If endpoint samples cannot
be compiled or compared, the row is `Unsupported` with a warning instead of
being silently omitted.

## Starter Template Gate

The automated tests require these controls to be at least `Clear`:

| Template | Controls |
| --- | --- |
| Sci-Fi Industrial Crate | Body Proportions, Structural Heft, Panel Depth, Vent Density, Handle Style, Detail Density |
| Roman Bridge | Span Length, Deck Width, Structural Heft, Support Rhythm, Bracing Style, Railing, Edge Finish |
| Stylized Lamp | Overall Height, Base Weight, Stem Curvature, Joint Size, Shade Style, Shade Scale |

The following controls may be `SubtleButExplainable` when the report labels
them honestly:

| Template | Controls |
| --- | --- |
| Sci-Fi Industrial Crate | Edge Softness |
| Stylized Lamp | Edge Softness |

## Determinism

Reports are sorted by control ID and use deterministic endpoint sample order.
Repeated runs over the same document and catalog must produce identical rows,
warnings, visible-delta scores, and perceptual reports.
