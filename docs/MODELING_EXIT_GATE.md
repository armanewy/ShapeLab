# Modeling Exit Gate

The modeling era passes only when Shape Lab can prove semantic modeling programs honestly.

## Forward Gate

- Rich asset families replay deterministically.
- Character and prop grammars use the same `shape-program` IR.
- Every supported operation declares topology effect, provenance, selection requirements, exact replay behavior, semantic parameter count, and inference hints.
- Blender adapters may cache or reorder for export, but cannot alter canonical semantic results.

## Inverse Synthetic Gate

- Synthetic hard-surface corpora recover exact semantic programs.
- Known-base character corpora recover exact compact programs.
- Success requires zero residual, zero literal target mesh bytes, and zero per-vertex independent positions.
- Equivalent programs may be accepted only when they replay to exact canonical topology and geometry and remain admissible.

## External Asset Gate

- External meshes are canonicalized with units, axes, transforms, manifold diagnostics, winding, part separation, and index preservation reported explicitly.
- The inverse compiler must rank known bases and grammars with confidence and rejection reasons.
- Partial explanations must report missing operators, unsupported topology, search limits, and residual diagnostics as failures.

## Product Gate

- Whole-model visual customization remains usable without Advanced Recipe for core tasks.
- Accepted candidates update visible controls rather than replacing the recipe opaquely.
- Exported packages include strict verification reports where applicable.

## Integrity Gate

No result may count as strict semantic success if it uses:

- literal target geometry
- opaque residuals
- dense arbitrary displacement
- per-vertex independent positions
- a target-derived base topology
- an export-only target-index permutation as semantic explanation

The correct outcome for an unsupported asset is an actionable failure report, not a hidden correction buffer.
